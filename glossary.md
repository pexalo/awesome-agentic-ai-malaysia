# Awesome Agentic AI Malaysia: glossary

Terms are defined for this repository. Other standards or organisations may use them differently.

## Agent

A software component that uses a model and instructions to decide what action to take toward a defined objective.

## Agent orchestration

The logic that assigns work, coordinates dependencies, manages handoffs, and handles failures across one or more agents.

## Agentic AI

AI systems designed to pursue goals through planning, tool use, state management, and iterative action rather than only generating one response.

## Evaluation

A repeatable process for measuring output quality, task completion, safety, latency, cost, and reliability.

## Guardrail

A technical or procedural control that limits actions, validates inputs or outputs, or requires approval.

## Human in the loop

A design in which a person reviews, approves, corrects, or escalates selected actions.

## Large language model

A model trained on large text corpora that can interpret and generate language and, in some systems, use tools.

## Memory

Information retained or retrieved to help an agent maintain context. Memory may be short-lived, persistent, or sourced from approved systems.

## Multi-agent system

A system in which multiple agents have distinct responsibilities and coordinate to complete broader work.

## Retrieval-augmented generation

A pattern that retrieves relevant source material and gives it to a model when producing an answer.

## Tool calling

A structured mechanism through which a model requests an approved function, API, search, or business-system action.

## Workflow automation

The execution of repeatable business steps using rules, integrations, and, where appropriate, AI-based decisions.
