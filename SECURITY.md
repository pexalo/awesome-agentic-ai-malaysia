# Security policy

## Supported material

This repository contains documentation, not production software. Security concerns may still involve malicious links, exposed secrets, unsafe examples, dependency or workflow configuration, private information, or claims that could lead to unsafe implementation.

## Reporting

Before public launch, the repository owner should enable GitHub private vulnerability reporting and publish an approved monitored security contact. Until that route exists, do not post sensitive details in a public issue. Use the current [Pexalo contact page](https://pexalo.com/contact) to request a private reporting channel without including exploit details or confidential data in the first message.

## Public corrections

Non-sensitive broken links, typos, and factual corrections may be reported through the contribution process after it is enabled. See [docs/security.md](docs/security.md) for the repository's broader security and governance guidance.
